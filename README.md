## 🎀 Постановка задачі

### Індивідуальне завдання №17
Визначити **8-річне та 16-річне** уявлення цілісного значення загального електричного опору трьох послідовно з'єднаних провідників при заданому постійному струмі та відомій напрузі на кожному провіднику.

### Три обов'язкові частини
Всі три виконані всередині одного проекту:

**Завдання 1** - серіалізований клас + агрегування. Розробити клас, що серіалізується, для зберігання параметрів і результатів обчислень. Використовуючи агрегування, розробити клас для знаходження рішення задачі. 

**Завдання 2** - діалоговий режим + transient поля. Розробити клас для демонстрації в діалоговому режимі збереження та відновлення стану об'єкта, використовуючи серіалізацію. Показати особливості використання transient полів. 

**Завдання 3** - тестування обчислень і серіалізації. Розробити клас для тестування коректності результатів обчислень та серіалізації/десеріалізації. Використовувати докладні коментарі для автоматичної генерації документації засобами javadoc.

## 🗂️ Архітектура класів

### `ResistanceData` - серіалізований клас даних
Зберігає всі параметри та результат обчислення.
Реалізує `Serializable` - об'єкт можна зберегти у файл і відновити.

| Поле | Тип | Опис |
|------|-----|------|
| `voltage1` | double | Напруга U1 |
| `voltage2` | double | Напруга U2 |
| `voltage3` | double | Напруга U3 |
| `current` | **transient** double | Струм - не серіалізується |
| `totalResistance` | double | Результат обчислення |

> 🔍 Поле `current` оголошено як `transient` - після відновлення з файлу
> воно скидається до `0`, що й демонструється у Частині 2.

---

### `ResistanceCalculator` - клас з агрегуванням
Містить об'єкт `ResistanceData` (відношення HAS-A - це і є агрегування).
Виконує обчислення та керує серіалізацією.

---

### `MainDialog` - діалоговий режим
Інтерактивне меню для роботи з програмою в консолі.

| Команда | Дія |
|---------|-----|
| `e` | Ввести напруги та струм |
| `c` | Обчислити опір |
| `v` | Переглянути дані |
| `s` | Зберегти у файл |
| `r` | Відновити з файлу |
| `q` | Вийти |

---

### `ResistanceTest` — тестування
Три незалежні тести без сторонніх бібліотек.

| Тест | Що перевіряє |
|------|-------------|
| Test 1 | Правильність обчислення R_total |
| Test 2 | Збереження даних після серіалізації |
| Test 3 | Скидання transient поля до 0 |

## 💡 Як працює програма?

Три провідники з'єднані **послідовно** - струм через усі однаковий.
За законом Ома `R = U / I`, тому:
```
R1 = U1 / I
R2 = U2 / I  
R3 = U3 / I
R_total = R1 + R2 + R3
```

Після обчислення `R_total` виводиться у **десятковій**, **8-річній** та **16-річній** системах числення.
