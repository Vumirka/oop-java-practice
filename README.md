### Індивідуальне завдання №17

### П'ять обов'язкових частин

**Завдання 1** - Як основа використовувати вихідний текст проекту попередньої лабораторної роботи. Забезпечити розміщення результатів обчислень уколекції з можливістю збереження/відновлення.

**Завдання 2** - Використовуючи шаблон проектування Factory Method (Virtual Constructor), розробити ієрархію, що передбачає розширення рахунок додавання нових відображуваних класів.

**Завдання 3** - Розширити ієрархію інтерфейсом "фабрикованих" об'єктів, що представляє набір методів для відображення результатів обчислень.

**Завдання 4** - Реалізувати ці методи виведення результатів у текстовому виде.

**Завдання 5** - Розробити тареалізувати інтерфейс для "фабрикуючого" методу.

## 💜 Про програму

Програма розширює попередній проект - тепер результати обчислень
зберігаються у колекції `ArrayList`. Для створення об'єктів відображення
використовується шаблон проектування **Factory Method**.
Колекція може бути збережена у файл та відновлена з нього.

## 🗂️ Архітектура - Factory Method

Шаблон Factory Method дозволяє створювати об'єкти не напряму,
а через фабричний метод - це робить ієрархію легко розширюваною.

| Роль | Клас/Інтерфейс | Опис |
|------|---------------|------|
| Creator | `Viewable` | Інтерфейс з фабричним методом `getView()` |
| ConcreteCreator | `ViewableResult` | Реалізує `Viewable`, створює `ViewResult` |
| Product | `View` | Інтерфейс з методами відображення |
| ConcreteProduct | `ViewResult` | Реалізує `View`, керує колекцією |

---

## 📦 Колекція результатів

`ViewResult` зберігає результати у `ArrayList<ResistanceData>`.
Кожен елемент - набір напруг, струм та порахований опір.

| Метод | Дія |
|-------|-----|
| `viewInit()` | Генерує 5 випадкових наборів даних |
| `viewShow()` | Виводить таблицю результатів |
| `viewSave()` | Серіалізує колекцію у файл |
| `viewRestore()` | Відновлює колекцію з файлу |
